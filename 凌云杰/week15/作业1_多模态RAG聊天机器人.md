# 作业1：多模态RAG聊天机器人需求分析与接口设计

## 一、需求分析

### 1.1 业务背景
基于Week15课程内容，需要构建一个多模态RAG（Retrieval-Augmented Generation）聊天机器人，支持对包含文本、图片、表格、公式等多种元素的复杂文档进行问答。

### 1.2 功能需求

| 需求编号 | 需求描述 | 需求来源 |
| :--- | :--- | :--- |
| R1 | 支持多种文档格式解析（PDF、图片、网页） | 05_jinai工具.md、02_MinerU.md |
| R2 | 支持多模态元素提取（文本、表格、图片、公式） | 02_MinerU.md |
| R3 | 文档向量化存储与检索 | 08_SearchRAG.py |
| R4 | 支持网络搜索作为补充知识库 | 08_SearchRAG.py、05_jinai工具.md |
| R5 | 深度研究能力（问题分解→多源检索→内容整合） | 09_DeepResearch.py |
| R6 | 支持图片理解（VLM能力） | 课程主题：ClaudeCode、文档解析 |

### 1.3 非功能需求

| 需求编号 | 需求描述 |
| :--- | :--- |
| N1 | 支持异步处理，提高并发性能 |
| N2 | 支持多种LLM模型接入 |
| N3 | 支持本地文档和网络资源混合检索 |
| N4 | 输出结构化结果，便于后续处理 |

│  └──────┬───────┘   └──────┬───────┘   └──────┬───────┘    │

│  ┌──────────────────────────────────────────────────────────┐       │

### 2.2 核心接口设计

#### 2.2.1 DocumentParser 接口

| 方法名 | 功能描述 | 参数 | 返回值 |
| :--- | :--- | :--- | :--- |
| `parse(file_path)` | 解析本地文档 | `file_path`: str - 文件路径 | `Document` 对象 |
| `parse_url(url)` | 解析网页 | `url`: str - 网页URL | `Document` 对象 |
| `extract_elements()` | 提取多模态元素 | 无 | `List[Element]` |

#### 2.2.2 VectorStore 接口

| 方法名 | 功能描述 | 参数 | 返回值 |
| :--- | :--- | :--- | :--- |
| `add_documents(docs)` | 添加文档 | `docs`: List[Document] | None |
| `similarity_search(query, k)` | 相似度检索 | `query`: str, `k`: int | `List[Document]` |
| `save(path)` | 保存索引 | `path`: str - 保存路径 | None |
| `load(path)` | 加载索引 | `path`: str - 加载路径 | None |

#### 2.2.3 SearchEngine 接口

| 方法名 | 功能描述 | 参数 | 返回值 |
| :--- | :--- | :--- | :--- |
| `search(query)` | 网络搜索 | `query`: str - 搜索词 | `List[SearchResult]` |
| `crawl(url)` | 抓取网页内容 | `url`: str - 网页URL | `str` - Markdown内容 |

#### 2.2.4 RAGChatbot 接口

| 方法名 | 功能描述 | 参数 | 返回值 |
| :--- | :--- | :--- | :--- |
| `add_document(file_path)` | 添加本地文档 | `file_path`: str | None |
| `add_url(url)` | 添加网页资源 | `url`: str | None |
| `query(question)` | 问答查询 | `question`: str | `Answer` 对象 |
| `deep_research(topic)` | 深度研究 | `topic`: str | `Report` 对象 |

## 三、数据结构定义

### 3.1 Document 类

| 字段名 | 类型 | 描述 |
| :--- | :--- | :--- |
| `id` | str | 文档唯一标识 |
| `content` | str | 文本内容 |
| `metadata` | dict | 元数据（标题、作者、日期等） |
| `elements` | List[Element] | 多模态元素列表 |
| `embedding` | Optional[List[float]] | 向量表示 |

### 3.2 Element 类（多模态元素）

| 字段名 | 类型 | 描述 |
| :--- | :--- | :--- |
| `type` | Literal["text", "table", "image", "formula", "header"] | 元素类型 |
| `content` | str | 内容（文本/表格HTML/图片描述/公式LaTeX） |
| `position` | Optional[dict] | 在原文档中的位置信息 |
| `metadata` | Optional[dict] | 附加信息 |

### 3.3 Answer 类

| 字段名 | 类型 | 描述 |
| :--- | :--- | :--- |
| `content` | str | 回答内容 |
| `sources` | List[Source] | 引用来源 |
| `confidence` | float | 置信度 (0-1) |
| `thought` | Optional[str] | 思考过程（用于调试） |

### 3.4 Source 类

| 字段名 | 类型 | 描述 |
| :--- | :--- | :--- |
| `type` | Literal["document", "web"] | 来源类型 |
| `url_or_path` | str | 来源路径或URL |
| `snippet` | str | 引用片段 |
| `relevance` | float | 相关性评分 (0-1) |

## 四、核心流程

### 4.1 文档加载流程

```
用户上传文档 → DocumentParser解析 → 提取多模态元素 → 
生成文本内容 → 计算向量嵌入 → 存入VectorStore
```

### 4.2 问答流程

```
用户提问 → 问题向量化 → VectorStore检索 → 
网络搜索补充 → 构建上下文 → LLM生成回答 → 返回结果
```

### 4.3 深度研究流程

```
用户主题 → 问题分解Agent → 多子问题并行检索 → 
内容抓取 → 章节起草 → 报告整合 → 返回完整报告
```

## 六、部署与集成

### 6.1 依赖环境

| 依赖名称 | 版本要求 | 用途 |
| :--- | :--- | :--- |
| Python | >= 3.10 | 运行环境 |
| langchain | >= 0.1 | 框架核心 |
| langchain-openai | >= 0.1 | LLM集成 |
| faiss-cpu | >= 1.7 | 向量存储 |
| mineru | >= 2.0 | 文档解析 |
| requests | >= 2.31 | HTTP请求 |
| aiohttp | >= 3.8 | 异步HTTP |

### 6.2 配置说明

```yaml
# config.yaml
llm:
  provider: dashscope
  model: qwen-max
  api_key: ${OPENAI_API_KEY}
  base_url: https://dashscope.aliyuncs.com/compatible-mode/v1

vector_store:
  type: faiss
  path: ./data/vector_store

search:
  jina_api_key: ${JINA_API_KEY}
  enabled: true

document_parser:
  mineru_enabled: true
  ocr_enabled: true
```

## 七、安全性考虑

| 风险点 | 应对措施 |
| :--- | :--- |
| 敏感信息泄露 | 对检索结果进行敏感信息过滤 |
| 恶意文档上传 | 限制文件类型和大小，进行病毒扫描 |
| 模型注入攻击 | 对用户输入进行清理和验证 |
| API密钥安全 | 使用环境变量存储，不硬编码 |
| 网络安全 | 使用HTTPS，验证外部API来源 |
