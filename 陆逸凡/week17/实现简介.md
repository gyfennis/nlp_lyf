## 2.SemanticCache 

工作流程可以概括为"先理解，再匹配，最后决定"：

1. **用户提问**：用户输入一个问题，比如"法国的首都是哪里？"
2. **向量化**：系统会立即将这个**问题**（注意，不包括答案）转换成一个向量——这个向量代表了问题的"语义指纹"。
3. **相似度搜索**：它拿着这个向量，去 Redis 中存储的**历史问题向量库**里进行搜索，找出语义上最相似的历史问题。
4. **阈值判断**：
   - **命中缓存**：如果找到的**最高相似度 ≥** 你设定的 `distance_threshold`，就认为命中。它会直接返回对应的历史答案，**整个过程完全不调用大模型**。
   - **未命中缓存**：如果最高相似度 < 阈值，就认为是新问题。它会调用大模型生成答案，然后把`问题-答案`对存进缓存，供下次使用。

### 核心参数：`distance_threshold`

`distance_threshold` 是你控制缓存"灵敏度"的关键旋钮。它基于**余弦距离**计算，取值范围是 `[0, 2]`。

| 阈值设置              | 效果                                                         | 适用场景                                                     |
| :-------------------- | :----------------------------------------------------------- | :----------------------------------------------------------- |
| **较低 (如 0.2)**     | **严格匹配**：只有意思几乎完全一样的问题才会命中缓存。       | **代码问答**：`"如何用Python读取CSV？"` 与 `"Python读取CSV的方法？"` 可能不命中，因为措辞差异较大。 |
| **中等 (如 0.5-0.7)** | **推荐值**：能找到"同一个意思，不同问法"的问题，是精确度和召回率的最佳平衡点。 | **通用客服**：`"我的订单什么时候到？"` 与 `"快递还要多久？"` 会命中。 |
| **较高 (如 0.9)**     | **宽松匹配**：只要意思大致相关就会命中，但可能误匹配，风险较高。 | **闲聊机器人**：`"今天天气真好"` 与 `"周末适合出去玩吗？"` 可能错误命中。 |

## 3. **EmbeddingsCache**

它使用的是**精确匹配**机制。它的设计目标非常明确：**避免重复为同一段文本生成向量**，从而为你省钱、提速。

它的工作流程就像一部精准的机器：

1. **生成唯一"指纹"**：当你准备为一段文本（比如"什么是机器学习？"）生成向量时，缓存会根据这段**文本内容** + **所用的模型名称**，通过哈希算法生成一个独一无二的ID。
2. **快速查找**：它会拿着这个ID去 Redis 里查找：
   - **命中**：如果能找到，说明这段文本用同一个模型算过向量了。它会立即把之前存好的结果返回给你，**完全绕过了昂贵的计算过程**。
   - **未命中**：如果没找到，它才会去调用真正的嵌入模型进行计算。得到结果后，它会以那个ID为钥匙，把新向量存进 Redis 里，供下次使用。

## 4.SemanticMessageHistory

专门用来管理**带语义理解能力**的聊天记录的,可以把它理解为一个 **“有记忆”并且能“举一反三”的聊天记录本**,能理解当前问题，并找回**意思相近**的历史记录，不论它在对话的哪个位置

- **核心参数 `distance_threshold`**：你可以把它理解为“相关度的灵敏度”。
  - **值越低 (如 0.3)**：要求越严格，只找意思几乎完全匹配的历史记录。
  - **值越高 (如 0.7)**：匹配越宽松，会找回更多意思相近的记录。

## 5.Semanticrouter

在代码里做 **意图识别与分发**。比如判断用户是想聊天、问技术还是查天气，然后交给不同的函数去处理。**简单语义匹配**。你预定义几个“路线”（比如政治、闲聊），每个路线配上一些例子，它通过计算向量相似度来判断输入属于哪一类。



# 达到的效果

实现了对大模型调用（LLM 对话、Embedding、意图识别等）过程的高效缓存，缓存组件可以实现按照 session id 分开存储对话，也可以缓存大模型历史的调用和回答。组件被用在多个项目中，减少了大模型和 Embedding 模型调用次数，并节约了计算资源。
