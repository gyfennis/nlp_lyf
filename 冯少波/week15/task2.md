# MinerU vs pdfplumber PDF 解析效果对比

## 一、测试环境

- **测试文档**: 深交所信息公司-毛瑞彬v1.5.pdf（33页，PPT风格文档，含图表/表格/图片）
- **pdfplumber 版本**: 最新版（基于 pdfminer.six + pypdfium2）
- **MinerU 版本**: 2.x（基于 PDF-Extract-Kit，含布局检测/OCR/公式识别/表格识别模型）

## 二、核心能力对比矩阵

| 对比维度 | MinerU | pdfplumber |
|---|---|---|
| **技术原理** | 多模块文档解析：布局检测 → OCR/公式识别/表格识别 → 后处理 → 格式转换 | 基于 pdfminer.six 提取 PDF 内嵌文本流 |
| **OCR 支持** | ✅ 内置 OCR，可处理扫描件、图像型PDF | ❌ 不支持，只能提取内嵌文本 |
| **版面布局分析** | ✅ 支持，可识别标题/正文/图片/表格/页眉页脚等区域 | ❌ 不支持，按文本流顺序输出 |
| **表格提取** | ✅ HTML 格式，支持无线表格、跨页合并、单元格合并 | ⚠️ `extract_tables()` 可提取有线表格，但无线表格/复杂表格效果差 |
| **图片提取** | ✅ 提取并上传至云存储，支持图片链接 | ✅ 可提取图片像素数据 |
| **Markdown 输出** | ✅ 直接输出结构化 Markdown | ❌ 纯文本，无格式标记 |
| **公式识别** | ✅ 支持 $LaTeX$ 公式识别 | ❌ 不识别 |
| **阅读顺序** | ✅ 基于布局分析恢复正确阅读顺序 | ❌ 多栏/PPT 等复杂布局乱序严重 |
| **跨页内容合并** | ✅ 支持跨页段落合并、跨页表格拼接 | ❌ 逐页独立提取 |
| **目录识别** | ✅ 支持自动识别目录结构 | ❌ 无此功能 |
| **中文支持** | ✅ 优秀，针对中文文档优化 | ⚠️ 取决于 PDF 内嵌字体编码 |
| **推理速度** | ⚠️ 较慢（需加载多个模型） | ✅ 极快（纯规则解析） |
| **GPU 需求** | ⚠️ 推荐 4GB+ VRAM | ❌ 不需要 |
| **资源占用** | 较大（模型约 20GB 磁盘 + 16GB 内存） | 极小 |

## 四、MinerU 论文核心思路（arXiv:2409.18839）

MinerU 是上海人工智能实验室（Shanghai AI Lab）开源的一站式文档提取工具，其核心思路：

### 4.1 技术架构

MinerU 采用**多模块文档解析策略（Multi-Module Document Parsing）**，分为四个阶段：

1. **文档预处理**（Document Preprocessing）
   - 筛选不可处理 PDF（加密、损坏等）
   - 识别语言类型（中/英文）
   - 检测是否为扫描件（基于图文面积比）
   - 检测字符乱码

2. **文档内容解析**（Document Content Parsing）
   - 使用 **PDF-Extract-Kit** 模型库
   - 布局检测：识别标题、正文、图片、表格、公式等区域
   - OCR：对文本区域进行文字识别
   - 公式识别：检测和识别行内/行间公式
   - 表格识别：检测和还原表格结构

3. **后处理**（Document Content Post-Processing）
   - 移除无效区域（页眉、页脚、页码等）
   - 基于位置信息拼接内容
   - 段落识别与合并（跨栏、跨页）

4. **格式转换**（Format Conversion）
   - 输出 Markdown 或 JSON 格式

### 4.2 关键创新

- **数据工程驱动的模型训练**：收集多样化 PDF 文档（学术论文/教材/试卷/研报等），聚类后标注训练
- **21K+ 多样化标注数据**：覆盖 10 种布局类型（标题/正文/图片/表格/公式等）
- **跨栏、跨页段落合并**：结合模型和规则的后处理
- **内容过滤**：自动过滤页眉页脚等干扰信息

## 五、总结

### 适用场景

| 工具 | 推荐场景 | 不推荐场景 |
|---|---|---|
| **pdfplumber** | • 纯文本 PDF 快速提取<br>• 简单有线表格提取<br>• 不需要版面分析的场景 | • 扫描件/图片型 PDF<br>• PPT/海报等复杂排版<br>• 需要保留标题结构的 RAG 场景 |
| **MinerU** | • 复合排版文档解析（研报/年报/教材）<br>• 需要 Markdown 结构化输出的 RAG 场景<br>• 含公式/复杂表格的学术文档<br>• 多种文档类型混合处理 | • 对速度要求极高的流水线<br>• 无 GPU 且文档结构简单的场景<br>• 轻量级部署 |

### 关键差异一句话总结

> **pdfplumber** 是轻量级的 PDF 文本流提取工具，无法处理复杂排版和扫描件；**MinerU** 是基于模型的完整文档解析系统，具备版面分析/OCR/表格/公式识别能力，能输出结构化 Markdown，但资源消耗大、速度慢。

### 在 RAG 项目中的建议

1. **简单 PDF（纯文本、单栏）** → 用 pdfplumber 即可，速度快、零依赖
2. **扫描件/图片型 PDF** → 必须用 MinerU 或类似 OCR 方案
3. **复杂排版（多栏/表格/公式）** → 推荐 MinerU，其 Markdown 输出可直接用于文档分块（Chunking），省去大量预处理工作
4. **混合策略**：先快速识别 PDF 类型，扫描件走 MinerU，简单 PDF 走 pdfplumber
